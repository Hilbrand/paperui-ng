openHAB - Empowering the smart home

This is a design study

What is this all about?

This is a (mostly non-functional) design study of a next generation Paper UI. Do not expect anything to work. This is about the design and presentation only. It should solve these issues:

  • A modern approach for Paper UI (technical reason).
  • A newby friendly presentation. A tour of all features. A guided tutorial.
  • All REST-API functionality should be covered, including Item Tags and Automation Rules.
  • Efficient multi item / batch editing features via textual view mode and multi item select.

This works best, if you actually connect to a running openHAB instance.
A word of warning: It is recommended not to use a production system. If you find any "remove" or "delete" buttons in this application, it might indeed "remove" and "delete" something.

How can I get involved?

Please post texts, pictures, videos of your ideas on how to improve the Paper UI interface to the Github Issue section of this project.
